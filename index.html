Blog article about Variable Fonts:

Did you know that before variable fonts existed, web developers had several different font files for static fonts? This was because each file was for the font’s weight, width, and their style combination. Some examples of these files included “'Roboto Regular', 'Roboto Bold', and 'Roboto Bold Italic',” just for the Roboto font (https://developer.mozilla.org). When combined, this would create one typeface displayed on the webpage. This was inconvenient for web developers because not only would they have to make separate files for each font style and weight, this would also put a strain on the web server as well because it would create many http requests, causing the webpage to slowly load. With the help of variable fonts, this allowed web developers to condense everything into one file and have access to all the information available. “As of May 2020 variable fonts are supported in most browsers.” (https://web.dev/variable-fonts/). This is important because when more browsers are supported with variable fonts, this allows them to become accessible to more users, allowing for a better user-friendly experience. The most common variable used is weight. The weight variable is connected to the bold style and depending on which location on the scale is chosen, the determines the type of bold style that will show on the typeface. This is determined by a point called an ‘instance’ on the scale that ranges from thin to thick. For example, “Weight axis location 600 is called SemiBold.” (https://web.dev/variable-fonts/). This may seem like a lot to the beginner web developer, but with many choices available, this allows them to get creative with their typeface. There are five main properties for variable fonts: weight, width, slant, optical size, and italics. Not all fonts contain all five of these values, so it is important to make sure to check and see if these properties are available for each font that you choose. If a web developer wants to make something artistic using these axes, then it would be best to experiment with either the title of the webpage, or a section that is not in the body. This is to prevent any risks with the readability of the webpage and still making sure that it fits within the accessibility guidelines. After reading this blog, you may ask yourself, how can I get started on using variable fonts? One website you can visit is axis-praxis.org. On there, you can play around with the different fonts, sizes, and widths to see what works best on your browser. Another website to play around with variable fonts is v-font.com.
